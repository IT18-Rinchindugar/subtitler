## Commands

```bash
yarn dev          # ReScript watch + Vite dev server (localhost:5173)
yarn build        # rescript compile + vite build
yarn vercel:build # res:clean + res:build + tsc + vite build (used in CI)
yarn test         # ReScript unit tests via retest
yarn res:build    # Compile ReScript only
yarn res:clean    # Clean ReScript build artifacts
```

Browser-based integration tests run via Vitest + Playwright (60s timeout for video processing). CI runs on every push/PR to `main` via GitHub Actions: install (frozen lockfile), res:build, tsc, format check, tests.

## Architecture

A fully client-side video subtitle tool: upload video → transcribe with Whisper (ONNX/WebAssembly) → edit cues → render subtitles onto video → export. No backend.

**Tech stack:** React 19 + ReScript 11 + TypeScript 5.4, Vite 5, Tailwind CSS 3.4, mediabunny (video codec), Konva (canvas), `@huggingface/transformers` (Whisper).

### ReScript / TypeScript split

Core logic lives in ReScript (`.res` files compile to `.res.mjs`); UI and React component glue lives in TypeScript/TSX. The bridge is `@genType`: annotating a ReScript type or function with `@genType` generates a `.gen.tsx` file that TypeScript can import directly. Generated files are checked in. Never edit `.gen.tsx` files — they are regenerated by `rescript`.

### State management

- **Editor state** — `EditorContext.res` holds a player reducer (`Play/Pause/Seek/SetDuration`) and a style reducer. Components subscribe via `useObservable` (a custom observable hook in `useObservable.res`) so only subscribed components re-render on updates.
- **Transcription state** — `useTranscriber.ts` manages Whisper model selection and inference; the model runs in a dedicated Web Worker (`transcriber/whisper-worker`). Model files are large (150 MB–700 MB) and cached in the browser after first download.
- **Persistence** — `useStickyState.res` syncs state to `localStorage` (model selection, language preference).

### Data flow

1. **Upload** — `LandingDropzone.tsx` accepts video; `LolApp.tsx` extracts AudioBuffer and dispatches it to the Whisper worker.
2. **Transcription** — Worker returns word-level chunks with timestamps. A pause gap >200 ms triggers a sentence boundary. Cues are auto-grouped by pause points and a 65-character soft limit.
3. **Editing** — `ChunksList/` renders the cue list; `ChunkEditor.res` handles per-cue text/timestamp edits. Pause boundaries are treated as immutable anchors — splitting/merging cues must respect them.
4. **Preview** — `EditorCanvas.tsx` + `subtitle-renderer.ts` render subtitles on a Konva canvas in real time. `active-word.ts` handles the word-highlight animation during playback.
5. **Export** — `render-worker.ts` (Web Worker) streams video frames, draws subtitles per-frame on canvas, then encodes with mediabunny (mp4/webm/mov). Uses File System Access API when available; falls back to blob download.

### Subtitle data model

- `subtitleCue` — `{ id, text, timestamp: (startSec, endSec | null) }` (defined in `Subtitles.res`)
- `wordChunk` — word-level timestamp record with pause index markers (defined in `WordTimestamps.res`)

### Configuration highlights

- `rescript.json` — outputs ES modules with `.res.mjs` suffix; JSX v4; genType enabled.
- `vite.config.ts` — React Compiler babel plugin (auto-memoization); PWA plugin with 10 MB workbox cache limit.
- `tailwind.config.cjs` — scans `*.tsx` and `*.res.mjs`; custom cinema color palette (amber/orange/gold/black); custom font families.
- Service worker is only registered in production; HMR is used in development.
